# ISO/IEC 42001 Conformance

> **Status**: Mapped — Annex A control alignment

## Overview

[ISO/IEC 42001:2023](https://www.iso.org/standard/81230.html) specifies
requirements for AI Management Systems (AIMS). CPoP addresses several
Annex A controls relevant to AI content detection and attestation.

## Annex A Control Mapping

| Control | Topic                        | CPoP Coverage                                                     |
| ------- | ---------------------------- | ----------------------------------------------------------------- |
| A.6     | Data governance              | Evidence data integrity via HMAC chains, WAL, and MMR append-only proofs |
| A.7     | System documentation         | Evidence packets include `claim_generator_info` with name, version, capabilities |
| A.8     | Transparency                 | Forensic verdict with confidence scores, anomaly counts, per-checkpoint flags, and `limitations` array |
| A.9     | Third-party relationships    | API consumer obligations documented; open protocol specification  |
| A.10    | Accountability               | Key hierarchy (master → session → ratchet) ties every action to a signing identity |

## A.8 Transparency Details

ISO 42001 control A.8 requires disclosure of system purpose, technical
limitations, and monitoring capabilities. CPoP Evidence Packets contain:

- **System purpose**: Protocol specification and claim_generator_info
- **Technical limitations**: `limitations` array (e.g., "keystroke data only; no semantic analysis")
- **Confidence levels**: `assessment_score` (0.0–1.0) with methodology (15+ forensic checks)
- **Monitoring**: Continuous sentinel process with checkpoint chain lifecycle
